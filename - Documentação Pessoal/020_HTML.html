divs, marcadores semânticos

DIVs = section article nav main header footer aside summary details === lógica para não usar sempre
div = agrupador de conteúdo
main = div para o conteúdo principal
header = div para o cabeçalho, inclui o conteúdo inicial
section = div com nome diferente
nav = divs com snaps, usando links para navegação
footer = div para o rodapé
section = seção autônoma, também é genérico como o div
article = composição independente em um documento, sub-artigos na página, jornais
aside = documento que nao faz parte do conteúdo, barras laerais por exemplo
figure = legendas para imagens por exemplo

Emmet

Auxiliar para digitar sintaxes de programação rapidamente
main>section>h1 = cria os elementos, um vinculado dentro do outro
h1+h2+h3 = cria os elementos um abaixo do outro
nav>ul>li*5>a[href=$]{Texto} = cria uma lista não ordenada de naveçaão com 5 itens enumerados
> = elementos vinculados
* = múltiplos elementos
+ = elementos adjacentes

Tabelas

Maneira clara e agradável de exibir alguns dados, dispondo de linhas e colunas
Elementos de tabela = table td tr th thead tbody tfoot colgroup caption
table = agrupa os elementos utilizados na tabela
td = célula de dados da tabela, apenas uma célula, coluna
tr = table row, linha de células na tabela, linha
th = cabeçalho da tabela
thead = marcador semântico para o cabeçalho de cima
tbody = marcador semântico para o conteúdo da tabela
tfoot = marcador semântico para o cabeçalho de baixo
rowspan = número de linhas ocupadas por um item na tabela
colspan = número de colunas ocupadas por um item na tabela

Formulários

form = separador que determina o formulário
action = para onde o dado é enviado
method = atributo que especifica como o dado será tratado
input = sem tag de fechamento, elementos de entrada, caixas de seleção, textos etcs
type = tipo de dado para entrada
placeholder = espaço reservado para indicar o que é a entrada
label = associado a entrada, para indicar o dado
id = dado, não repetir as variáveis
for = vincula a label ao id
button = solicitação, fora do formulário inicialemnte não faz nada, dentro do formulário realiza a ação se não especificado
name = tipo de dado enviado, rotula o dado
textarea = campo para texto
min max minlength maxlength = tamanho máximo para número e textos
required = campo obrigatório
checkbox = caixa de seleção
radio = caixa de escolha, no caso de somente ser possível uma opção dentre várias
select + option = caixa de seleção com texto flutuante
pattern = expressão regular, com um padrão como em senhas que exigem números por exemplo
usar divisões para o formulário não ficar confuso, div, p, etc
