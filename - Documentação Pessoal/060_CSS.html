seletores etc
selector {property: value;}
seletor pode selecionar um conjunto de elementos, por exemplo h1{color:purple;}

seletor universal
*{color: black;} --- seleciona tudo e deixa preto

seletor de elementos
button{font-size: 30px;} --- todos os botões terão o tamanho da fonte de 30px

seletor de elementos combinados
h1,h2{color: black;} --- todos os elementos de h1 e h2 serão pretos

seletor de id
#button1{color: white;} --- o elemento com o id "button1" será branco
se usar um seletor de elementos e um seletor de id, todos os elementos vinculados a partir do id terão as características fornecidas para o id, mas os elementos sem id especificado no seletor terão a característica especificada para o elemento geral
IDS NÃO PODEM SER REPETIDOS

seletor de classe
.vermelhos{color: red;} --- todos os elementos da classe vermelhos recebem a cor vermelha
segue a mesma lógica dos ids, os elementos da classe tem prioridade sobre os sem identificação
classes "genéricas", para vários elementos que serão daquele jeito, porém sem usar excessivos ids

seletor descendente
body nav a {color: yellow;} --- elementos "a" dentro de "nav" dentro de "body" receberão a cor amarela

seletor adjacente (combinadores) "+"
h2 + button {color: red} --- os botões no mesmo nível após h2 receberão a cor vermelha

seletor descendente direto ou (filho direto)
footer > a {color: white} --- o elemento "a" (hyperlink) que vem após o footer DIRETAMENTE (DIRECT CHILD) recebe a cor branca

seletor de atributo
input[type="password"] {color: green} --- os inputs do tipo senha receberão a cor verde
a[href*="google"] --- hyperlinks que tem a palavra google, no começo ou no final
ATENÇÃO, uma classe não é um atributo

Pseudo classes (:)
tipos de modificadores, palavras chaves, adicionadas ao final de um seletor para especifica um estado particular dos elementos selecionados, mouse por cima

IDs geram particularidades quando se tenta usar as pseudo classes, atenção

hover active ckecked nth-of-type(Xn)
hover checked first first-child last-child hover not() nth-child() nth-of-type()

a:hover{color: black;} --- "a"(hyperlinsk) mouse por cima, muda a cor do texto para preto
button:active{background-color: purple;} --- muda a cor dos botões quando ativados
input[type="checkbox"]:checked{box-shadow: 0 0 0 3px orange;} --- muda a sombra quando checkboxs estão marcadas
.vermelhos:nth-of-type(2n){background-color: white} --- a cada 2 itens, o primeiro terá a cor de fundo branca [nth-of-type(2) --- apenas o segundo item tera a cor de fundo branca]

seletores podem ser combinados, exemplo seletores de classes com seletores descendentes
.vermelhos a {color: darkred} --- todos os elementos "a" (hyperlinks), dentro da classe "vermelhos" receberão a cor vermelho escuro

Pseudo elementos (::)
modificadores dos elementos, estilizar a primeira letra ou primeira linha por exemplo

::after
::before
::first-letter
::first-line
::selection

h2::first-letter{font-size: 30px;} --- a primeira letra de todos os elementos h2 terá tamanho de 30px
p::first-line{color: red;} --- a primeira linha de todos os parágrafos será vermelha
p::selection{background-color: black; color: orange;} --- quando selecionados, os elementos do parágrafo serão da cor laranja com o fundo preto

cascada
quando existem múltiplos elementos para o mesmo elemento, o último é o que prevalece, segue uma ordem de cascada, vai carregando primeiro os de cima depois os debaixo, e o debaixo é o que efetivamente carrega na página
exemplo
h1{color: yellow;}
h1{color: black;}
a cor de h1 será preta, pois foi o último a ser carregado, mantendo a característica de cor preta

especificidade
somente é importante quando propriedades geram conflitos
mais de um estilo aplicável no mesmo elemento, gerando "erros"
exemplo:
.botoes button:hover{background-color: red;}
button:hover{background-color: blue;}
todos os botões terão o fundo azul quando pairados, menos os botões da classe "botoes", pois possuem uma classe definida, mantendo assim prioridade aos elementos quando a cor entra em conflito
outro exemplo:
p{color: yellow;} --- seletor de elementos
section p{color: blue;} --- seletor de elementos + seletor de elementos (prioridade)

Ordem de prioridade:
ID > CLASSE > ELEMENTO
0 0 0 --- quanto mais específico, maior a prioridade
estilos dentro do código html são mais específicos que ids, mas não é recomendado usá-los

no inspecionar do google na parte de css, dá pra ver as propriedades que estão efetivamente fazendo alterações em um elemento, e também visível as outras que aplicam algum tipo de conflito, porém estão "perdendo" por ser menos específica ou por vir em cima

a excessão !important
usado após definir o valor da propriedade, é o mais específico possível
não recomendado

se definirmos uma cor para o body, e depois uma cor para o paragrafo, a do paragrafo irá prevalecer por ser mais específica, alguns elementos não herdam cores por padrão por exemplo, como botões e inputs de texto, existe um comando para isso que é, no caso do botão, button{color: inherit;}, assim ele herdará a cor da sua seção por exemplo, não é muito comum de ser utilizado

Ordem de prevalência com todos os elementos de especificidade
!important > dentro do código html > id > classe > elemento

Display(exibição)
inline elemente and block elementos
inline elements fit in alongside other elements
block level elements take up a whole "block" of space

inline block inline-block
h1{display: inline;} --- elements like h1 (block level) now are inline elements on display
span{display: block;} --- elements like span (inline elements) now are block elements on dsplay
inline elements doesn't respect width and height spacementes, just horizontal that they are aligned
block elements not overlap another elements
inline-block elements are inline elements that respects width and height properties
var{display: none;} --- oculta elementos

in google dev tool "inspect", the content is represented by the blue color, padding by the green color, border by the "yellow" color, and margin by the orange color

unidades
tamanhos definidos por
relativos: em, rem, vh, vw, %, outros
absolutos: px, pt, cm, in, mm
PX absoluta mais comum, não é exatamente um pixel, varia de acordo com o monitor não recomendado em sites responsivos
mms in usados pra imprimir etc, não são comuns
unidades relativas são as comumente usadas
% - porcentagem de alguma coisa, é a porcentagem do elemento pai dependendo da utilização
exemplo: h1{font-size: 40px; line-height: 200%;} --- a altura da linha é o dobro do tamanho da fonte

EM - relaciona com o elemento pai
exemplo: h2{font-size: 2em;} --- h2 tem o dobro do tamanho da fonte de seu elemento pai, se estiver dentro de um div com tamanho da fonte de 40px por exemplo, ele terá 80px de tamanho da fonte, definido por 2em (dobro do elemento pai), comum utilizá-los em margins, paddings, etc, muito úteis para escalar o tamanho dos elementos conforme onde é utilizado
um erro que acontece comumente, é como em listas dentro de listas o tamanho vai aumentando conforme o nível de ligações (primeira lista respeita o elemento pai anterior as listas, porem as próximas listas em diante, relacionarão o tamanho com o tamanho da linha anterior), ficando cada vez maior/menor

REM - relativo ao "root html element"
elemento raíz html
html{font-size: 20px;}
se o elemento raíz tiver 20px, 1rem sempre terá 20px e 2 rem 40px, não varia conforme "graus de ligações"

algumas outras propriedades
alpha channel (canal alpha)
rgba altera a cor do fundo
rgba(255, 0, 0, 1) --- opacidade 100%, alpha varia de 0 a 1
#FF0000FF --- opacidade 100% em hexadecimal, alpha varia de 00 a FF

opacity (opacidade)
var{opacity: 0.5} --- 50% de opacidade, varia de 0 a 1 (0~100%)
opacidade altera a cor de todos os elementos da seção, não só a cor de fundo como o rgba





position (posição)
define como um elemento é posicionado na página
valores podem ser estáticos, relativos, fixos, absolutos
static: top right bottom left z-index não tem efeito, é o valor padrão, estático
var{position: static;}

relative: top right bottom left
var{position: relative; top: 100px;} --- offset de 100 px na parte superior

absolute
posicionado em relação ao elemento ancestral mais próximo
ou posicionado em relação ao bloco inicial
var{position: absolute; top: 10px; left: 10px;} --- offset de 10px à esquerda e em cima do início do "ancestor element", o qual ele é relacionado

fixed
posicionado em relação a topo da página, é como um absolute porém fixo mesmo quando há scroll na página, e não depende de elemento ancestral
var{position: fixed;}

sticky
é um fixed que pode ser alterado, para deixar de ser fixo em determinado momento
var{position: sticky;}





transition (transições)
existem diversos comandos para todas as formas de transição possível, consultar MDN
property-name duration timing-function delay
animar a propriedade de um elementopara outra propriedade
quando utilizado um hover de cor por exemplo, é possível adicionar um tempo de transição para que a cor mude de fato, em javascript é possível fazer uma integração muito mais complexa
var{transition: background-color 1s ease-in, border-radius 2s 1s} --- as 4 funções em um código

property-name + duration (s, ms)
define o que mudará com delay, e todo o resto não definido mudará instantaneamente no caso de um hover por exemplo
var{transition: background-color 1s;} --- somente a cor de fundo tera delay de 1s para se alterar
é possível adicionar um delay para que o delay comece
var{transition: background-color 2s 1s;} --- ao pairar o botão por 1s, a animação de 2s irá começar
var{transition: all 750ms 250ms} --- todas as propriedades definidas no hover e no objeto normal serão alteradas após pairar por 250ms, uma transição com a duração de 750ms
var{transition: background-color 1s, border-radius 2s;} --- a cor de fundo demora 1s e o raio demora 2s, começando a contar a partir do momento em que o elemento é pairado

timing-function
linear ease-in ease-out steps cubic-bezier
var{transition-timing-function: linear;} --- transição linear
var{transition-timing-function: ease-in;} --- vai ficando mais rápido
var{transition-timing-function: ease-out;} --- vai ficando mais devagar
var{transition-timing-function: steps(6,end);} --- número de passos
var{transition-timing-function: cubic-bezier(.29, 1.01, 1, -0.68);} --- "curva de aceleração"





transform
consultar MDN, imensas funcionalidades
girar, distorcer, esticar, dimensionar, enviar, mover, dentre outras várias outra coisas

rotate (girar)
rotate() rotate3d() rotateX() rotateY() rotateZ()
unidades: deg grad rad turn
graus(circulo = 360º) gradianos(circulo = 400º) radianos(circulo =2π) turn(circulo = 1 turn)
deg
origem de rotação por padrão é o centro do elemento, mas é possível trocar
var{transform-origin: bottom-right} --- define a origem de rotação do canto inferior direito
var{transform: rotate(180deg);} --- define que o elemento será girado 180º

scale (escala)
mudar o tamanho do elemento proporcionalmente, para mais ou para menos
scale() scale3d() scaleX() scaleY() scaleZ()
var{transform: scale(0.5);} --- 50% do tamanho original
var{transform: scale(2, 1);} --- largura 200% do tamanho original e largura igual ao original

translate(mover)
mover elementos na página, usando pxs, rems, ems, etc
mover horizontalmente X (+ = direita / - esquerda), mover verticalmente Y (+ = baixo / - = cima)
translate() translate3d() translateX() translateY() translateZ()
var{transform: translateX(20px);} --- move 20px para a direita
var{transform: translate(-200px, 100px)} - move 200px para a esquerda e 100px para baixo

skew(distorcer)
distorce um elemento 2d
skew() skewX() skewY()
unidades: deg grad rad turn
var{transform: skew(30deg);} --- 30º em X e Y
var{transform: skew(10deg, 25deg);} --- 10º em X e 25º em Y

é possível combinar elementos de transformações
var{transform: rotate(90deg) scale(1.30);} --- rotacionado em 90º e 130% do tamanho original
var{transform: translateX(-500px) rotate(0.5turn);} --- movido 500px para a esquerda e de cabeça para baixo
afeta todos os elementos dentro da seção que está sendo transformada
matrix e perspectiva são comandos mais complexos

flexbox e css responsivos
flexbox's (layouts flexíveis)
parte mais recente do css, responsável por layout de páginas em caixas, a medida que a janela é proporcionada a orientação pode mudar por exemplo, de alinhamentos horizontais para alinhamentos verticais
display: flex
2 eixos, main axis(principal) da esquerda pra direita, cross axis(cruzado) de cima pra baixo (reverses inverte a direção) os eixos são manipulados de acordo com o eixo principal


direção do eixo principal
flex-direction (direção dos elementos no eixo principal)
row(padrão) row-reverse column column-reverse
direção dos elementos na página a partir dos 2 eixos

alinhamento no eixo principal
justify-content (alinhamento principal, baseado no eixo principal)
flex-start flex-end flex-center flex-between flex-arround space-evenly
o começo e o fim é baseado na orientação dos elementos, se começarem na direita, o fim será a esquerda

direção no eixo cruzado
flex-wrap (utiliza uma disposição baseada no eixo cruzado, modifica sua direção)
wrap wrap-reverse nowrap

alinhamento no eixo cruzado
align-items (distribui os itens no eixo cruzado)
flex-start flex-end center baseline(linha de baixo do texto)

espaço entre os elementos no eixo cruzado
align-content (distribuir o espaço ao longo do eixo cruzado, controle de espaço entre o eixo cruzado)
space-between space-arround flex-start flex-end center
somente funciona quando há wrap criando elementos ao longo do eixo cruzado

alinhamento ao longo do eixo cruzado em um único elemento
align-self (elemento individual no eixo cruzado baseado no espaço disponível no eixo cruzado)
flex-end flex-start center

width ou height ao longo do eixo principal (height em column, width em row)
flex-basis (define o tamanho inicial do elemento antes de colocá-lo no contêiner)
px em rem etc

preenchimento de acordo com o espaço disponível
flex-grow (preenchimento baseado no espaço livre disponível)
1 dígito, em caso de mais de um elemento com essa característica, o número funciona como uma proporção (por ex, elemento 1 com grow 1 e elemento 2 com grow 2, o elemento 2 ocupa o dobro do tamanho do elemento 1 quando são redimensionados para um tamanho maior)
pode ser definito um limite de tamanho em (max-width e max-height, unidades em em rem px etc)

velocidade de diminuição do elemento
flex-shrink (diminuição baseado no espaço que os elementos ocupam)
0 = não encolhe, um dígito, quando maior, mais rápido encolhe quando redimensionado

atalhos de digitação
flex (atalho de digitação para vários comandos de flex, ordem de todas as possíveis no MDN)
var{flex:1;} --- flex grow
var{flex:1 600px} --- grow basis
var{flex:1 2 600px} --- grow shrink basis





design responsivo
nas ferramentas de inspeção do google é possível visualizar a versão para dispositivos móveis
site funcional e bonito em todos os dispositivos, mobiles, monitores mais largos, resoluções diferentes, etc, é possível utilizando media queries
media queri permite modificar o estilo dependendo os parâmetros particulares dos dispositivos
principal mecanismo para reestilizar um site responsivo de acordo com a orientação, dispositivo etc

@media (media queries)
widhth height orientation baseados no viewport(o que é visível)
@media(min-width: 800px){h1{color: purple;}} --- acima de 800px a cor de h1 é roxo
media queris podem ser combinadas
@media(min-width: 600px) and (max-width: 800px){h1{color: purple;}} -- entre 600px e 800px de largura da página, h1 é roxo
a ordem de valores importa no código em CSS, valores menores devem estar em cima no código
em códigos de max-width um de 1000px e um de 500px, o de 1000px deve setar em cima do de 500px no código
@media(orientation: landscape){h1{background-color: magenta;}} --- se a orientação for paisagem, terá um fundo de cor magenta em h1
max-width --- valores abaixo do definido ativarão a condição
min-width --- valores acima do definido ativarão a condição

os scripts do bootstrap são colocados nas últimas linhas de código antes de </body>
as classes pré-definidas nos comandos do bootstrap são criadas no código de link usado no <head>
a maioria dos comandos em bootstrap é utilizado com as classes que estão nos comandos

container --- fundamental em um sistema de grades
elemento mais básico do bootstrap, utilizado para adequar o sistema de grade
container redimensiona a exibição da página de acordo com o tamanho da visualização

btn --- classe de botões
primary secondary success danger warning info light dark link etc são customizáveis
depende do código a ser usado, toda documentação está escria no site do bootstrap, assim como o MDN

display --- tamanho de exibição
tamanho de exibição, elementos h1 h2 h3 etc precisam ser consecutivos, não é o caso dos displays
útil em elementos de títulos, subtítulos

lead --- destacar parágrafos
deixa os parágrafos um pouco maiores, mudando o tamanho da fonte e a largura da fonte

blockquote puro(html) é usado para citações, uma linha menor que a normal
blockquote --- destacar citações
blockquote-footer --- nome de quem fez a escrita original da citação por ex

mb (margem) --- define margem para os elementos

alignment utilities (alinhamento usando bootstrap) --- alinhamento do texto
text-right text-left text-center

colors --- cores
text-primary text-info etc

background colors --- cores de fundo
bg-secundary bg-warning etc

badge --- títulos, botões, etc
escalado de acordo com o elemento pai (h1 h2 h3 etc)
badge bagde-primary etc

button group --- agrupa vários botões
botões dentro de um div, depois agrupados usando um button group
role="group" --- indica que é um grupo de botões para leitores de tela

alerts --- feedback ao usuário, salvo, falha, bem vindo de volta etc
mostrar ao usuário alguma coisa importante
class="alert" alert-primary (classe + cor) etc
alert-heading --- título para botões
dimissing × --- fechar alerta (utiliza javascript)
roles="alert" class="close" data-dismiss="alert"

aria-laber aria-hidden --- auxiliares para leitores de tela





grid system (layout)
ajuda a fazer o layout na página utilizando bootstrap
só funciona dentro de um container, precisa de um ou mais containers na página

criar a row usando a classe row
cada row no bootstrap tem 12 unidades (SEMPRE)
criar uma coluna ussando a classe col-U (U é o número de unidades usadas da row)
col sem definir unidades, divide o espaço igualmente entre o número de elementos
col-sm-8 --- quando a exibição for considerada "pequena"(documentação no bootstrap), o o elementos empilham, mas quando for maior que isso, ocupará 8 unidades
no-gutters --- elimina os espaços entre elementos em colunas
img-fluid --- responsive images
img class="col"

comandos de alinhamento de conteúdo etc, são escritos após a classe row
alinhamento utilizando bootstrap (flexboxes in bootstrap)
row align-items-end align-items-start align-items-center
align-self-start align-self-end align-self-center --- align self in bootstrap

justify content
justify-content-start justify-content-end justify-content-center justify-content-between

border --- cria uma borda

forms
form-control --- melhor visualização para entrada de dados
form-control-sm lg etc --- tamanho do input
custom-control custom-checkbox --- personalizações para checkboxes são mais complexas
custom-radio custom-range
custom-switch --- checkbox switch
form-row --- espaço mais compacto, melhor para formulários
col-md-6 --- abaixo da resolução "média", eles vão empilhar em duas linhas
col-md-6 col-4 --- abaixo da média ocupa 6 unidades, acima disso ocupa 4
